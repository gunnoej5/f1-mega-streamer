# F1 Mega Streamer

F1 Mega Streamer is a desktop utility for Formula 1 fans who want to watch multiple F1TV streams in standard browser windows while keeping playback synchronized. The utility provides centralized controls for playback, seeking, and stream management instead of embedding the video experience inside the application.

## Project Purpose

This repository is both:

- the working project space for F1 Mega Streamer as a standalone public GitHub repository, and
- a documented feedback artifact for TheCubicles factory workflow.

The intent is to manage the product in a normal repo shape while preserving the planning, review, and operational outputs that came out of the factory run. That gives future development a clean home and creates a useful loop back into TheCubicles: what worked, what broke, what needed hardening, and what should be improved in the factory process itself.

## What Is In This Repo

- `src/`: application source code across backend, frontend, and desktop packaging layers
- `plan/`: requirements, user stories, beta remediation planning, and beta signoff criteria
- `review/`: review findings and recommendations from the generated deliverable
- `ops/`: release and monitoring artifacts for packaging, deployment, and beta operations
- `MANIFEST.json`: factory run metadata and gate history for the generated deliverable

## How To Use This Repo

- Use `plan/requirements.md` and `plan/user-stories.md` as the product baseline.
- Use `plan/beta-readiness-plan.md` as the execution plan for remediation and beta prep.
- Use `plan/beta-readiness-plan2.md` as the final go/no-go checklist before publishing a beta build.
- Use `review/code_review.md` as the source of truth for the known findings that need remediation.
- Use `ops/` artifacts when packaging, smoke testing, monitoring, and releasing builds.

## Feedback Loop To TheCubicles

This repo is meant to do more than hold code. It should also capture the delta between factory output and production reality.

Examples:

- product planning refined after the initial generated deliverable
- code review findings that exposed correctness or security gaps
- beta remediation plans that show what a factory output still needed before release
- release and monitoring artifacts that can improve future factory-generated ops outputs

That makes the repository a practical feedback loop: develop here, document what had to be fixed, then feed those lessons back into TheCubicles to improve future runs.
